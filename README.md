# AI OS MVP (личная) — Next.js + Supabase

Это «полноценный MVP» под твои требования:
- Логин обязателен (Supabase Auth)
- Дашборд показывает ВСЕ ИИ-сервисы + комментарии (на русском)
- Дашборд показывает сценарии (сайт/бот/реклама/стратегия/рынок/продукт/софт) — запуск создаёт новый проект
- Каждый запуск создаёт «доску» (Kanban) + базу данных проекта (ключи данных со статусами) + шаги
- В шагах промпты генерируются автоматически из заполненных данных (тебе не нужно подставлять вручную)

## Что внутри
- `/` — дашборд
- `/projects` — все проекты
- `/projects/new?type=site&mode=A|B` — создание проекта
- `/projects/[id]` — доска проекта
- `/projects/[id]/data` — данные проекта (статусы + формы + «промпт для сбора»)
- `/projects/[id]/steps/[stepId]` — шаг: автопромпт + сохранение результата

## 1) Локальный запуск
### Требования
- Node.js 18+
- Supabase проект

### Установка
```bash
npm i
```

### ENV
Скопируй `.env.example` → `.env.local` и заполни:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY

### База данных (Supabase SQL Editor)
Запусти по очереди:
1) `sql/001_init.sql`
2) `sql/002_seed_scenarios.sql`

### Запуск
```bash
npm run dev
```
Открой `http://localhost:3000`

## 2) Деплой (Vercel)
1) Создай GitHub репозиторий, залей код.
2) В Vercel: New Project → Import → Deploy.
3) Vercel → Settings → Environment Variables:
   - NEXT_PUBLIC_SUPABASE_URL
   - NEXT_PUBLIC_SUPABASE_ANON_KEY
4) Redeploy.

## 3) Примечание по «автоматизации»
- Автоподстановка работает через `project_data` (то, что ты заполнил в «Данные проекта»).
- На шаге ты копируешь уже готовый промпт → открываешь нужный сервис → вставляешь ответ обратно.
  Это MVP-уровень автоматизации без сложной интеграции API.
